How KAVACH Works

From deployment to detection in 4 simple steps. Deploy invisible honeypots, catch attackers the instant they move.

🐳

$ docker pull kavach/server:latest

$ docker run -d -p 8080:8080 kavach/server

✓ Dashboard ready

✓ API active

Step 1

Deploy

Run KAVACH in Docker on any infrastructure. Single command. Takes 5 minutes. Zero complexity.

  • ✓ Works on any cloud (AWS, Azure, GCP, on-prem)
  • ✓ Self-hosted — your data stays yours
  • ✓ Zero external dependencies
  • ✓ No vendor access to your systems
  • ✓ Go binary — fast, lightweight, reliable

Step 3

Place Strategically

Hide tokens where attackers look during reconnaissance. They're invisible to legitimate users but irresistible to intruders.

Communication Channels

Infrastructure

Network shares, file servers, S3 buckets

Source Control

Private repos, CI/CD secrets, deployment configs

The 7-Dimensional Risk Classifier

Every access to a honeypot is analyzed across 7 dimensions to produce a 0–100 risk score. ML-ready architecture for continuous improvement.

1. IP Reputation

25%

Private vs public IP analysis, known bad IP lists, VPN/proxy detection, Tor exit node identification

2. Request Rate

15%

Requests per minute, scanning behavior patterns, DoS/DDoS indicators, brute force detection

3. Payload Analysis

15%

SQL injection detection, XSS patterns, command injection attempts, large/malformed payload analysis

4. Header Fingerprint

12%

Missing standard headers, bot/automation signatures (curl, wget, Python), custom user-agent analysis

5. Behavioral Anomaly

12%

Path traversal attempts, admin/sensitive path access, null byte injection, hidden file access patterns

6. Geolocation

12%

VPN indicators, unusual geographic access, data residency violations, multi-hop proxy detection

7. Timing Patterns

9%

Machine-like consistency, non-human request intervals, automation detection, off-hours activity
